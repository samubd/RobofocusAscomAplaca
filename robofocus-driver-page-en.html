What is the Robofocus ASCOM Alpaca Driver

The Robofocus ASCOM Alpaca Driver is a modern, full-featured driver for controlling Robofocus electronic focusers through the ASCOM Alpaca v1 protocol.

This driver represents a cutting-edge solution for astrophotography, enabling seamless integration of your Robofocus focuser with major imaging software like N.I.N.A. (Nighttime Imaging 'N' Astronomy), SGP, APT, and any other ASCOM Alpaca-compatible software.

Why ASCOM Alpaca?

ASCOM Alpaca is the new HTTP/REST-based communication standard that replaces the old COM protocol. It works on any platform, is more reliable, and allows remote access to your focuser from different devices on your local network.

Key Features

  • Fully Open Source: Source code available on GitHub, developed with Python and FastAPI
  • Zero-Config Installation: Windows installer that automatically configures everything, including firewall
  • Hardware + Simulator: Works with real Robofocus hardware or in simulation mode for testing
  • Web Control Panel: Modern web interface for manual control and configuration
  • Auto-Discovery: NINA and other software automatically detect the driver on the network
  • Reliable and Robust: Error handling, automatic retries, intelligent position caching

How to Use

The Robofocus driver can run on Windows or Linux systems. NINA and other clients can connect to it from any machine on your network.

Installation - Choose Your Platform

Windows Installation (Recommended for NINA users)

1 Download the installer from the Download section (file RobofocusAlpaca_Setup_x.x.x.exe)
2 Run the installer and follow the setup wizard. The installer will automatically configure:
  • The driver in the C:\Program Files\RobofocusAlpaca folder
  • Shortcuts in Start Menu and Desktop
  • Windows Firewall to enable automatic discovery (UDP port 32227)
3 Launch the driver from the Start Menu or Desktop shortcut
4 Configure the driver by opening your browser at http://localhost:5000
Alternative: Use the portable version (RobofocusAlpaca_Portable.zip) if you don't want to install. Just extract and run RobofocusAlpaca.exe.

Linux Installation (Headless server or Raspberry Pi)

1 Install Python 3.8+ (usually pre-installed on most distributions) 
# Debian/Ubuntu
sudo apt update
sudo apt install python3 python3-pip python3-venv

# Fedora/RHEL
sudo dnf install python3 python3-pip

# Arch Linux
sudo pacman -S python python-pip
2 Clone the repository and navigate to it 
git clone https://github.com/samubd/RobofocusAscomAplaca.git
cd RobofocusAscomAplaca
3 Create a virtual environment (recommended) 
python3 -m venv venv
source venv/bin/activate
4 Install dependencies 
pip install -r requirements.txt
5 Run the driver 
python3 -m robofocus_alpaca

The server will start on http://0.0.0.0:5000 (accessible from other machines on the network)

Serial Port Permissions: Add your user to the dialout group to access serial ports without sudo: 
sudo usermod -a -G dialout $USER
# Logout and login again for changes to take effect
Run as a Service: To run the driver automatically on boot, create a systemd service: 
sudo nano /etc/systemd/system/robofocus-alpaca.service
Add: 
[Unit]
Description=Robofocus ASCOM Alpaca Driver
After=network.target

[Service]
Type=simple
User=yourusername
WorkingDirectory=/path/to/RobofocusAscomAplaca
ExecStart=/path/to/RobofocusAscomAplaca/venv/bin/python -m robofocus_alpaca
Restart=always

[Install]
WantedBy=multi-user.target
Enable and start: 
sudo systemctl enable robofocus-alpaca
sudo systemctl start robofocus-alpaca
sudo systemctl status robofocus-alpaca
Firewall Configuration: If you have a firewall enabled, allow incoming connections on port 5000 (HTTP) and 32227 (UDP discovery): 
# UFW (Ubuntu/Debian)
sudo ufw allow 5000/tcp
sudo ufw allow 32227/udp

# Firewalld (Fedora/RHEL)
sudo firewall-cmd --permanent --add-port=5000/tcp
sudo firewall-cmd --permanent --add-port=32227/udp
sudo firewall-cmd --reload

Initial Configuration

(Same for both Windows and Linux - use the web interface)

On first run, the driver will guide you through the configuration:

  1. Choose mode:
    • Hardware: to use your real Robofocus connected to the serial port
    • Simulator: to test without hardware (ideal for configuring NINA before going to the field)
  2. Select COM port (Hardware mode only):
    • Click "Scan" to automatically detect your Robofocus
    • Or manually select the COM port from the list
  3. Connect by clicking the "Connect" button
  4. Calibrate the focuser (if needed):
    • Set the zero reference point
    • Configure maximum extension
    • Set backlash (if your model supports it)

Using with NINA

  1. Start the Robofocus driver
  2. Open NINA
  3. Go to Equipment ‚Üí Focuser
  4. Click Alpaca Discovery (NINA will automatically detect the driver)
  5. Select Robofocus from the list
  6. Click Choose ‚Üí Connect
  7. Test functionality with "Move to" button or run Autofocus
Tip: If automatic discovery doesn't work, you can manually add the address http://localhost:5000 in NINA.

Web Panel Features

The web control panel (http://localhost:5000) offers:

  • Status Dashboard: Current position, temperature, connection status, active mode (Hardware/Simulator)
  • Quick Step Controls: Buttons for fast movements (¬±1, ¬±10, ¬±100, ¬±1000 steps)
  • Absolute Positioning: Input field to go to a specific position with limit validation
  • Emergency HALT: Immediate stop button to halt any ongoing movement
  • Calibration:
    • Set Zero: defines current position as zero reference
    • Set Max Extension: sets maximum travel limit (saved to hardware)
    • Set Min Position: sets minimum limit (software-side)
    • Set Max Increment: limits maximum single move (safety)
    • Set Backlash: configure backlash compensation (¬±255 steps, saved to hardware)
  • Protocol Logs: Real-time view of serial commands sent/received, with filters and statistics

Support and Troubleshooting

Driver doesn't appear in NINA

  • Verify that the driver is running
  • Check Windows Firewall (UDP port 32227 must be open)
  • Try manually adding http://localhost:5000 in NINA
  • Check logs in robofocus_alpaca.log

Timeout errors during movement

Since version 1.0, this issue has been resolved with an automatic settling delay (150ms) after each movement. If it persists:

  • Verify that the serial cable is properly connected
  • Check that the COM port is correct
  • Try reducing the focuser movement speed

COM port not found

  • Verify that the Robofocus is powered on and connected
  • Check in Windows Device Manager for the COM port number
  • Use the "Scan" button in the Settings page to automatically detect
  • Some USB-to-Serial adapters require specific drivers
